# ASF Progress Notes — 2026-05-28

## Summary

Work completed since the last Claudio call covers four areas: (1) two correctness bugs found and fixed in the ASF pipeline and benchmark infrastructure, (2) a new routing flag `ASF_ALWAYS_STAGE25` that forces DeBERTa to run on every input, (3) a full multi-dataset benchmark suite with proper per-sample isolation across 10 datasets and 7 pipeline configurations, and (4) two follow-up items from Claudio's feedback — a documentation context dampener in L1.5 and an FP analysis confirming the 15.4% FPR was entirely a bug.

## Bug Fixes

### 1. ZERO_WIDTH_RANGES false positive (ASF — commit 4febb64)

The `ZERO_WIDTH_RANGES` list in `hardening.py` included the range `(0x0000, 0x000F)`, which covers `\n` (0x0A), `\t` (0x09), and `\r` (0x0D). Any multi-line input was being flagged as containing zero-width attack characters, causing widespread false positives.

Fix: split the range into `(0x0000, 0x0008)`, `(0x000B, 0x000C)`, `(0x000E, 0x000F)`, explicitly excluding common whitespace characters.

Impact on OPI (67,200 samples): 5,160 false positives eliminated. L1.5 recall dropped from 82.5% to 50.0% — the previous number was inflated by legitimate multi-line benign content being flagged. **The 15.4% FPR Claudio raised is gone: confirmed 0 FPs on 33,600 OPI benign samples (see FP analysis below).**

### 2. Benchmark env-var isolation bug (eval — commits fd96ca7, bcf3772)

All benchmark scripts built a `benchmark_rows` list at module level:

```python
benchmark_rows = [
    ("ASF L1.5 only",        run_config(samples, classify_l15, ...)),
    ("ASF Stage 1+2",        run_config(samples, classify_s12, ...)),
    ...
]
```

Each `classify_*` function called `configure_env(...)` before classification, but Python evaluates all `run_config(...)` calls sequentially at parse time, so every call used the `os.environ` state left by the last `configure_env` call. All pipeline variants were silently running with identical settings.

Fix: moved `configure_env(...)` inside each `classify_*` function so it applies per sample at runtime.

This affects all pipeline stage results other than L1.5-only and ONNX (which do not use `configure_env`). Those two rows were always correct.

## New Feature: ASF_ALWAYS_STAGE25 (commit 0e47de0)

**Problem**: Semantic attacks score near zero on L1.5 heuristics and are cleared by the `HEURISTIC_CLEAR` fast-path (threshold ≤ 0.05) before reaching DeBERTa. `ASF_SOFT_THRESHOLD` was intended to escalate borderline cases, but semantic attacks score ~0 so they never exceed it.

**Solution**: new flag `ASF_ALWAYS_STAGE25=true` that:

1. Bypasses the `HEURISTIC_CLEAR` fast-path, so every input reaches Stage 1+2.
2. Escalates Stage-2-SAFE verdicts to DeBERTa (Stage 2.5a), not just UNCERTAIN.

**Observed effect**:

| Dataset | Without Always-Stage25 | With Always-Stage25 |
|---|---|---|
| deepset (direct injection, 546) | recall 11.3%, FPR 1.2% | recall 100%, FPR 94.5% |
| jackhhao (jailbreaks, 1044) | recall 16.3% | recall 99.6%, FPR 96.3% |
| llm-semantic-router (jailbreaks, 2480) | recall 3.5% | recall 65.2%, FPR 59.2% |
| bipia (indirect injection, 5000) | recall 0.5% | recall 100%, FPR 99.8% |

**Finding**: DeBERTa detects the attacks effectively but over-triggers on benign content at rates that make it unusable as a standalone gate. The FPR reflects DeBERTa being tuned for injection detection, not general safe/unsafe classification. Forcing it on everything is not a production-viable routing strategy — a smarter escalation signal is needed (see Open Items).

## Responses to Claudio's Feedback

### FPR 15.4% (feedback point 3.2)

Confirmed as entirely the zero-width bug. After the fix: **0 FPs on 33,600 OPI benign samples** (`benchmarks/analyze_fp.py`, commit e361945). The documentation context dampener (below) was added proactively for document-heavy inputs, but was not needed to reach 0% FPR on OPI.

### Documentation context dampener (hardening.py — commit be88a4b)

Claudio suggested reducing FP risk on document-structured inputs (headers, bullet points, code blocks). Implemented as `_detect_document_context(text) -> float` in `hardening.py`. When document confidence ≥ 0.4, the `instruction_lang` and `entropy` feature scores are multiplied by `(1 - doc_confidence)` before the final heuristic score is computed.

Attack-specific features (`zero_width`, `structural`, `known_payloads`, `sensitive_file_abuse`, `base64`, `rot13`) are not dampened. Escape hatch: `ASF_DISABLE_DOC_DAMPENER=true`. The `doc_context` score is included in `ClassifierResult.features` for audit visibility.

### Sigil baseline (feedback point 3.3)

Noted. After the zero-width fix, the comparison is: ASF L1.5 **50.0% recall / 0.0% FPR** vs Sigil v2 **53.5% recall / 0.0% FPR** on OPI. A 3.5pp gap at equal FPR. The `summary_table.py` cross-dataset view uses the correct post-fix numbers.

### L1.5 as only decision-maker (feedback point 3.1)

Partially addressed via `ASF_ALWAYS_STAGE25`. Full resolution requires a smarter routing signal — routing all HEURISTIC_CLEAR inputs directly to DeBERTa is too aggressive (see Open Items).

### deepset sample count (feedback point 3.5)

Our run uses 546 samples. Claudio's run uses 662 (`split='train'`). To align: update `benchmark_deepset.py` to load from HuggingFace with `split='train'` instead of the local JSON. Not yet done.

## Tooling

**`benchmarks/summary_table.py`**: reads all `*_results_full.json` files and prints a per-dataset table (Section 1) and a cross-dataset recall matrix for four key configs (Section 2). `--markdown` flag writes `benchmarks/summary.md` as GFM tables.

**Union classifier**: all 10 benchmark scripts include `classify_union` (`classify_l15 OR classify_onnx`) and an `"ASF L1.5 + ONNX (union)"` row. Results will populate on next run of each script.

**`benchmarks/analyze_fp.py`**: runs `classify_text` on all OPI benign samples and groups false positives by intent and triggering feature. Result as of 2026-05-28: `n_fp=0, fpr=0.0`.
